# docs 目录说明

本目录用于沉淀**对外可复用**、**可追溯**的文档资料，避免信息散落在聊天记录或临时笔记中。

## 维护者

- 维护者：AI（Codex）

## 目录结构约定（强制）

- `docs/plan/`：任务/需求拆解与执行计划（目标、约束、步骤、验证方式、回滚方案）
- `docs/architecture/`：架构与设计决策（模块边界、关键流程、协议/数据结构、权衡取舍）
- `docs/progress/`：进展与里程碑记录（已做、已验证、未解决问题、下一步）
- `docs/reference/`：稳定参考资料（配置项说明、API/协议摘要、常用命令/约定、外部依赖对接要点）

## 命名规则

- 优先使用 `YYYY-MM-DD-主题.md`（便于按时间回溯）。
- 若同一天多份文档，可追加序号：`YYYY-MM-DD-主题-01.md`。

## 何时需要更新

- 新增/变更对外契约（CLI 命令、参数、输出格式、配置项）时必须同步更新 `docs/reference/`。
- 任务开始、关键节点完成、出现阻塞/风险时必须更新 `docs/progress/`。
